Detailed Flow

Step 1: User creates a request 📝

A signed-in user opens the form and provides all details required for fast team triage.

  • Title and detailed problem description.
  • Deadline, requester type, and contact details.
  • Required skills and languages.
  • Automatic link to the user's account.

Example:

Title: We need mentoring for our hiring process
Category: Hiring
Skills: recruiting, hiring, operations
Deadline: within 10 days

Step 2: Internal team triages the request 🔍

A super-admin or worker opens the request in the dashboard and decides the next action.

  • Validates completeness of the request.
  • Sets priority and category.
  • Adds review notes and chooses next step.
  • Workers are limited to their assigned categories.

Admin sets:

  • Priority: High
  • Category: Hiring
  • Impact: 9/10
  • Status: In Review

Step 3: The system proposes expert matches 🧠

Matching creates a shortlist, but final assignment stays under team control.

  • Skills: overlap between request and expert profile.
  • Languages: alignment with required languages.
  • Experience text: relevance from bio and work history.
  • Rating: reputation signal from completed work and feedback.

Example shortlist:

Anna Nováková

84% match based on skills, languages, and experience

Step 4: Team assigns experts and starts coordination 💬

Once an expert is assigned, the request enters active work and all key communication remains in-system.

  • Admin assigns one or more experts.
  • Experts are marked busy when they have active tasks.
  • Request chat captures request-specific coordination.
  • Internal details can be handled in company/admin chats.

After assignment:

The request moves into active resolution,
the expert appears in assigned tasks,
and both requester and internal team can track progress in one place.

Step 5: Expert delivers the work 🤝

During execution, experts can clarify context, consult with stakeholders, and coordinate next steps.

  • Works with request details and deadlines.
  • Uses chat for clarification and alignment.
  • Can leave a request if unable to continue.
  • Help is tied to a concrete outcome, not only conversation.

Possible outcomes:

  • Mentoring or expert consultation.
  • Operational support on assigned work.
  • Closed hiring, sales, or networking request.
  • Clear ownership of delivery.

Step 6: Requester confirms completion 📊

The final step is performed by the request creator. This distinguishes delivered help from requests still in progress.

  • The request is marked as done/resolved.
  • The system stores the completion timestamp.
  • The expert receives +1 karma and help credit.
  • Busy status is recalculated based on active tasks.

Recorded in the system:

Status: Done

Resolution: Request closed by requester

Credit: Expert receives karma and help count update

Expert: Anna Nováková

Process Summary

The workflow combines automated recommendations with human decisions and closes only when the requester confirms the result.
