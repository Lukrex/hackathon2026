📚 API Documentation

Kompletný REST API pre Community Help System. JSON responses, filtrovanie, triedenie, pagination.

🌍 API Base URL

http://localhost:8000/api/

Súčasť Endpoints

📝 Requests API

Správa žiadostí o pomoc

List all requests

GET

/api/requests/

Vylistuj všetkých žiadosti s možnosťou filtrovania, triedenia a paginovania.

Query parameters:

?status=open&priority=high&search=React&ordering=-created_at&page=1

Príklad response:

{ "count": 5, "results": [ { "id": 1, "title": "Need React Developer", "priority_display": "🔴 Vysoká", "status_display": "🟡 V preverovaní", "created_at": "2026-03-28T10:30:00Z" } ] }

Get request details

GET

/api/requests/{id}/

获取detaily o jednej konkrétnej žiadosti.

Create request

POST

/api/requests/

Vytvor novú žiadosť o pomoc.

Request body:

{ "title": "Need senior frontend developer", "description": "Details about the requirement", "requester_name": "John Doe", "requester_email": "john@example.com", "requester_phone": "+421123456789", "category": 1 }

Assign expert

POST

/api/requests/{id}/assign_expert/

Priraď experta k žiadosti.

{ "expert_id": 1 }

Mark as resolved

POST

/api/requests/{id}/mark_resolved/

Označ žiadosť ako vyriešenú.

{ "notes": "Successfully hired developer John Smith" }

👥 Experts API

Správa a prehliadanie expertov

List experts

GET

/api/experts/

Adresár všetkých expertov s filtovaním a vyhľadávaním.

Query parameters:

?availability=high&search=React&ordering=-help_provided

Get expert profile

GET

/api/experts/{id}/

Detaily o jednom expertovi.

🧠 Matches API

Expert-request zhodnosti a scoring

List matches

GET

/api/matches/

Vylistuj všetky zhodnosti.

?request=1&ordering=-match_score

🏷️ Categories API

List categories

GET

/api/categories/

Všetky dostupné kategórie žiadostí.

Status Codes

200 OK

Successful GET request

201 Created

Successful POST request (resource created)

400 Bad Request

Invalid request data

404 Not Found

Resource doesn't exist

Response Format

Všetky responses sú v JSON formáte.

Successful Response (200/201)

{ "id": 1, "title": "Request title", "description": "Description", "status": "open", "priority": "high", ... }

Error Response (4xx/5xx)

{ "error": "Error message", "detail": "More detailed error information", "field_errors": { "email": ["Invalid email format"] } }

Filtrovanie a Triedenie

Requests filtrovanie:

status=open

Filter by status (open, in_review, in_progress, resolved, rejected)

priority=high

Filter by priority (critical, high, medium, low)

category=1

Filter by category ID

search=React

Full-text search

Triedenie:

/api/requests/?ordering=-created_at # Najnovšie prvé /api/requests/?ordering=priority # Podľa priority /api/experts/?ordering=-help_provided # Podľa počtu pomôcí (klesajúco)

Príklady použitia (cURL)

GET all requests

curl http://localhost:8000/api/requests/

POST create request

curl -X POST http://localhost:8000/api/requests/ \ -H "Content-Type: application/json" \ -d '{ "title": "Need DevOps engineer", "description": "Looking for experience with AWS and Kubernetes", "requester_name": "Jane Doe", "requester_email": "jane@example.com", "category": 1 }'

GET expert matches

curl http://localhost:8000/api/requests/1/suggested_experts/

POST assign expert

curl -X POST http://localhost:8000/api/requests/1/assign_expert/ \ -H "Content-Type: application/json" \ -d '{"expert_id": 2}'

Autentifikácia

API je momentálne verejný (no login required pre čítanie).
V produkcii môžeš pridať Token-based authentication.

💡 Tipy

  • API vracia JSON. Nastav Accept: application/json header
  • POST/PUT requesty vyžadujú Content-Type: application/json
  • Pagination vracia count, next, previous, results
  • Use ordering=field pre ASC a ordering=-field pre DESC
  • Kombinuj filtre: ?status=open&priority=high&search=React
